Linux AI Assistant

Command-line AI assistant for Linux that executes bash commands, searches the web, and works with local LLMs (Ollama, LM Studio) or cloud APIs.

⚠️ Safety first – Every command requires your confirmation before execution.
Features

    🤖 OS automation – Describe what you want, AI generates bash commands

    🧠 Local or cloud LLMs – Ollama, LM Studio, OpenAI, Anthropic, etc.

    🔍 Auto web search – DuckDuckGo for factual questions

    🔐 Sudo mode – Optional prefix for commands

    📜 Command logging – Saves to ~/command_log.txt

    💻 Manual fallback – !bash <command> runs directly

Requirements

    Linux

    Python 3.8+

bash

pip install requests duckduckgo-search

Optional per provider:

    Ollama (port 11434) with pulled model

    LM Studio (port 1234) with server running

    API key + base URL (OpenAI/Anthropic/compatible)

Setup

    Save as linux_ai_assistant.py

    Run: python3 linux_ai_assistant.py

    Follow prompts for:

        Sudo mode (y/n)

        AI provider (1=Ollama, 2=LM Studio, 3=API)

        Model selection

Usage

Just type what you want, e.g.: update apt

The AI outputs a bash command in a code block – you confirm before execution.

Commands:

    exit – quit

    /toggle – toggle auto web search

    /search <query> – force search

    !bash <command> – run bash directly

⚠️ Recommended Models

For reliable Linux automation, use large enough models (7B+ parameters) or cloud models. Small models (<7B) typically lack the reasoning and command-generation accuracy for multi-step OS tasks.

Recommended:

    Cloud: GPT-4, Claude 3.5+, GPT-3.5-Turbo

    Local (7B+): Llama 3.x (8B+), Qwen 2.5 (7B+), Mistral (7B+)

    Local (larger): Llama 3 70B, CodeLlama 34B

Small models will produce incorrect bash commands or fail at chaining.
Notes

    Commands are logged to ~/command_log.txt

    Sudo mode = assistant may delete/modify system files – confirm carefully

    Requires internet for web search
